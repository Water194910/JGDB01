# 视觉处理方案 — maixcam2 + STM32F407

## 一、概述

视觉系统负责靶心检测、距离估算、激光点标定，为云台控制提供高质量的目标偏移数据。数据链路：

```
maixcam2 (YOLO检测 + 图像处理) → UART → STM32F407 (云台控制)
```

### 需要解决的问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 视觉原始数据有噪声 | YOLO检测框抖动、光照变化 | 卡尔曼滤波 |
| 同样偏1像素，远近对应角度不同 | 透视投影 | 逆透视变换 + 距离补偿 |
| 激光点与相机光轴不重合 | 安装误差 | 激光点位标定 |
| 控制增益需要随距离调整 | 远目标需要更大修正量 | 动态放大系数 |

## 三、逆透视变换

### 3.1 为什么需要

相机光轴与云台激光束存在安装角度偏差。靶纸在画面中不是正对相机，而是有透视倾斜。同样大小的物理偏移，在画面不同位置对应的像素数不同。

```
透视畸变示意：

  实际靶纸（正方形A4）          画面中看到的（梯形）
  ┌──────────────┐              ╱‾‾‾‾‾‾‾‾‾‾‾╲
  │              │             ╱              ╲
  │      ●       │            │      ●        │
  │    (靶心)    │            │    (靶心)      │
  │              │             ╲              ╱
  └──────────────┘              ╲__________╱

  靶纸偏左1cm → 画面上方偏5px，下方偏8px（透视近大远小）
```

### 3.2 原理

通过已知的4个标定点（A4纸的4个角），计算从画面像素坐标到实际物理坐标的映射矩阵。

```
标定步骤：
1. 将A4纸放在靶位，画面中识别到4个角点
2. 记录4个角点的像素坐标 (u1,v1)~(u4,v4)
3. 已知A4纸实际尺寸：210mm × 297mm
4. 计算透视变换矩阵 H（3×3单应矩阵）

运行时：
  对每个检测到的目标中心 (cx, cy)，应用 H 变换
  → 得到校正后的坐标 (cx', cy')
  → dx = cx' - laser_x', dy = cy' - laser_y'
```

### 3.3 实现方式

**maixcam2 端**（推荐）：
- 标定时计算 H 矩阵，存为常量
- 每帧检测后对目标中心做透视变换
- 优点：STM32 收到的 dx/dy 已经是校正后的，控制端简单

**标定 H 矩阵的计算**（OpenCV 习惯）：

```python
import numpy as np

# 画面中4个角点的像素坐标（顺时针，从左上开始）
pts_pixel = np.float32([[u1,v1], [u2,v2], [u3,v3], [u4,v4]])

# 对应的实际物理坐标（mm，以靶心为原点）
pts_real = np.float32([[-105, 148.5], [105, 148.5], [105, -148.5], [-105, -148.5]])

H = cv2.getPerspectiveTransform(pts_pixel, pts_real)

# 运行时变换
pt = np.float32([[cx, cy]])
result = cv2.perspectiveTransform(pt.reshape(1, -1, 2), H)
cx_real, cy_real = result[0][0]  # 物理坐标 mm
```

> maixcam2 基于 Linux，可用 OpenCV 或 numpy。如果不方便用 OpenCV，也可以在 maixcam2 上手动实现 3×3 矩阵乘法。

## 四、激光点位标定

### 4.1 为什么需要

激光发射器和相机安装位置不同，激光打在靶上的位置 ≠ 相机光轴中心。视觉系统需要知道"激光现在打在哪"，才能计算"激光离靶心还有多远"。

```
  画面中：
  ┌────────────────────┐
  │                    │
  │      ● 靶心        │
  │         ↖          │
  │          ◆ 激光点   │
  │                    │
  └────────────────────┘

  dx = 靶心cx - 激光x
  dy = 靶心cy - 激光y
```

### 4.2 标定方法

**方法一：静态标定（简单）**
1. 云台不动，激光常亮
2. 相机拍到激光点在画面中的像素坐标 (laser_x, laser_y)
3. 将这个坐标写死到视觉代码中
4. 每帧计算偏移时用 `dx = cx - laser_x`

**方法二：动态标定（更准）**
1. 在 maixcam2 上增加激光点检测逻辑
2. 激光点特征：高亮度、小面积、红色/绿色
3. 每帧同时检测 A4 靶纸和激光点
4. `dx = a4_cx - laser_cx, dy = a4_cy - laser_cy`

**方法三：光轴对齐（最理想）**
1. 物理调整相机和激光的安装位置，使激光点尽量靠近画面中心
2. 激光点 ≈ (half_w, half_h)
3. 此时 `dx = cx - half_w, dy = cy - half_h`（即当前代码的方案）
4. 残余偏差通过软件标定补偿

### 4.3 建议

先用**方法一**（静态标定，写死坐标），验证整个链路跑通。后续如果精度不够再升级到方法二或方法三。

## 五、卡尔曼滤波

### 5.1 在哪滤波？

| 位置 | 优点 | 缺点 |
|------|------|------|
| **maixcam2 端（推荐）** | 噪声源头处理，发给STM32的数据已干净 | maixcam2 已在跑YOLO，需确认CPU余量 |
| STM32 端 | 控制环内实时滤波 | STM32 已在跑PID，增加计算负担 |

**建议在 maixcam2 端滤波。** 原因：
1. 噪声来自 YOLO 检测框的帧间抖动，在源头滤掉最干净
2. 发出去的 dx/dy 已经平滑，STM32 不需要额外处理
3. maixcam2 的 CPU（K210/V831）跑 YOLO 后通常还有余量做简单滤波
4. 减少 UART 传输的毛刺，降低通信出错概率

### 5.2 滤波什么？

对 **dx、dy** 两个量分别做一维卡尔曼滤波（或简化的一阶低通）。

也可以对 **检测框的 (x, y, w, h)** 做滤波，然后再算 dx/dy——这样框更稳定，距离估算也更准。

### 5.3 简化方案：一阶低通

如果卡尔曼实现复杂，先用一阶低通替代，效果对云台够用：

```python
# 一阶低通滤波
alpha = 0.3  # 0~1, 越小越平滑
dx_filtered = alpha * dx_raw + (1 - alpha) * dx_filtered_prev
dy_filtered = alpha * dy_raw + (1 - alpha) * dy_filtered_prev
```

> α=0.3 对应截止频率约 6Hz（假设检测帧率 30Hz），保留追踪运动，滤掉检测抖动。

### 5.4 完整卡尔曼方案

如果需要更好的效果，用匀速模型卡尔曼：

**状态向量：** `[x, y, vx, vy]`（位置+速度）

**状态转移：**
```
x'  = x + vx × dt
y'  = y + vy × dt
vx' = vx
vy' = vy
```

**观测：** `[x_meas, y_meas]`（YOLO 检测到的 dx/dy）

**过程噪声 Q：** 较小（目标运动平滑）
**观测噪声 R：** 较大（YOLO 检测有抖动）

卡尔曼会自动在"相信预测"和"相信观测"之间取平衡。当目标匀速运动时预测准，跟得紧；当检测出野值时自动抑制。

```python
# 伪代码
class Kalman2D:
    def __init__(self):
        self.x = np.zeros(4)  # [x, y, vx, vy]
        self.P = np.eye(4) * 100
        self.Q = np.eye(4) * 0.1   # 过程噪声
        self.R = np.eye(2) * 10    # 观测噪声
        self.H = np.array([[1,0,0,0],[0,1,0,0]])
        self.F = np.array([[1,0,dt,0],[0,1,0,dt],[0,0,1,0],[0,0,0,1]])

    def predict(self):
        self.x = self.F @ self.x
        self.P = self.F @ self.P @ self.F.T + self.Q

    def update(self, z):
        y = z - self.H @ self.x
        S = self.H @ self.P @ self.H.T + self.R
        K = self.P @ self.H.T @ np.linalg.inv(S)
        self.x = self.x + K @ y
        self.P = (np.eye(4) - K @ self.H) @ self.P

    def get_filtered(self):
        return self.x[0], self.x[1]  # dx, dy
```

## 六、距离估算

### 6.1 原理

A4纸物理宽度固定 210mm。通过画面中框的像素宽度 w_pixel 反推距离：

```
D = W_real × f / w_pixel
```

其中 f 是焦距（像素单位），通过标定获得。

### 6.2 标定焦距

```python
# 将A4纸放在已知距离 D_cal（如1000mm），记录框的像素宽度 w_cal
f = w_cal * D_cal / 210.0
# f 是常数，写死到代码中
```

### 6.3 运行时计算

```python
D_mm = 210.0 * f / a4.w   # 当前距离 mm
D_m  = D_mm / 1000.0       # 转换为米
```

### 6.4 用途

1. **动态放大系数**：远目标需要更大的角度修正
2. **调试显示**：串口输出距离，方便观察
3. **未来扩展**：根据距离调整射击参数等

## 七、动态放大系数

### 7.1 为什么需要

视觉 dx/dy 是像素偏移。同样 10 像素的偏移：
- 目标在 1m 处 → 实际偏移小 → 需要小角度修正
- 目标在 3m 处 → 实际偏移大 → 需要大角度修正

如果用固定系数，近处修正过猛（振荡），远处修正不足（追不上）。

### 7.2 公式

```
kv_dynamic = kv_base × (D / D_ref)

D = 当前估算距离
D_ref = 参考距离（标定时的距离，如1.5m）
kv_base = 参考距离处的基础系数
```

| 距离 | kv_dynamic | 效果 |
|------|-----------|------|
| 0.5m (近) | 0.33 × kv_base | 修正柔和 |
| 1.0m | 0.67 × kv_base | 适中 |
| 1.5m (参考) | 1.0 × kv_base | 基准 |
| 3.0m (远) | 2.0 × kv_base | 修正激进 |

### 7.3 限制范围

```c
kv_dynamic = kv_base * (D / D_ref);
if (kv_dynamic < kv_min) kv_dynamic = kv_min;  // 防止近处太弱
if (kv_dynamic > kv_max) kv_dynamic = kv_max;  // 防止远处太猛
```

## 八、协议扩展

### 8.1 当前协议（8字节）

```
[0] 0xAA  [1] 0xBB  [2] cmd  [3-4] dx  [5-6] dy  [7] checksum
```

### 8.2 扩展协议（10字节）

增加框的像素宽度 w（用于距离估算）：

```
[0]    0xAA        帧头1
[1]    0xBB        帧头2
[2]    cmd         0=扫描, 1=追踪
[3-4]  dx          int16 LE, 像素偏移X（已滤波+逆透视校正）
[5-6]  dy          int16 LE, 像素偏移Y
[7-8]  w           uint16 LE, 框的像素宽度（用于距离估算）
[9]    checksum    字节[2..8]的累加和 & 0xFF
```

STM32 侧收到后：
```c
float D = 210.0f * f_cal / (float)w;       // 距离 mm
float kv = kv_base * (D / D_ref);           // 动态系数
float delta_yaw  = (float)dx * kv;          // 角度增量
float delta_pitch = (float)dy * kv;
zhou_yaw.target_angle   -= delta_yaw;
zhou_pitch.target_angle += delta_pitch;
```

## 九、maixcam2 代码改造要点

当前 main(2).py 的改造清单：

| 改造项 | 说明 | 优先级 |
|--------|------|--------|
| 卡尔曼/低通滤波 | 对 dx/dy 做滤波后再发 | ★★★ 必做 |
| 协议扩展 | 多发 w 字段（框宽度） | ★★★ 必做 |
| 逆透视变换 | 标定 H 矩阵，校正 dx/dy | ★★☆ 推荐 |
| 激光点标定 | 记录 laser_x/y，偏移计算更准 | ★★☆ 推荐 |
| 距离计算 | maixcam2 端算好 D，直接发距离 | ★☆☆ 可选 |

### 建议实施顺序

1. **先加滤波 + 扩展协议**（最简单，效果最明显）
2. **再加激光点标定**（静态标定即可）
3. **最后加逆透视**（需要标定流程，稍复杂）

## 十、STM32 侧改动

云台方案中需要配合修改的部分：

### 10.1 协议解析

更新视觉帧解析，从 8 字节改为 10 字节，提取 w 字段。

### 10.2 动态系数计算

```c
float D = 210.0f * f_cal / (float)w_pixel;
float kv_dynamic = kv_base * (D / D_ref);
CLAMP(kv_dynamic, kv_min, kv_max);
```

### 10.3 Flash 参数扩展

```c
typedef struct {
    // ... 现有参数 ...
    float f_cal;        // 焦距标定值 (像素单位)
    float kv_base;      // 基础放大系数
    float D_ref;        // 参考距离 mm
    float kv_min;       // 动态系数下限
    float kv_max;       // 动态系数上限

```

### 10.4 调试指令扩展

| 指令 | 格式 | 说明 |
|------|------|------|
| 焦距 | `FC 1428` | 设置焦距标定值 |
