# 底盘控制方案 — MSPM0G3507

## 一、方案总览

MSPM0G3507 (TI) 作为底盘主控，负责循迹、差速转向平滑控制，以及向云台 STM32 发送横摆角速度前馈数据。

### 核心职责

1. **循迹运动**：沿固定轨迹行驶
2. **差速转向平滑**：左右轮速度按梯形/S形曲线渐进变化，避免剧烈转弯给云台制造难以补偿的晃动
3. **横摆角速度前馈**：读取底盘 BNO085 IMU，计算 yaw_rate，通过 UART 发送给云台 STM32

### 硬件清单

| 部件 | 型号 | 接口 | 说明 |
|------|------|------|------|
| 主控 | MSPM0G3507 | — | 在底盘上 |
| 底盘IMU | BNO085 | UART, RVC模式 | 底盘重心位置安装 |
| 左轮电机 | — | PWM/DRV | 差速控制 |
| 右轮电机 | — | PWM/DRV | 差速控制 |
| 循迹传感器 | — | GPIO/ADC | 循迹线检测 |

### 与云台的接口

MSPM0 通过滑环将横摆角速度数据发给云台 STM32（UART4 RX）。10字节二进制帧，~100Hz。

---

## 二、底盘IMU — BNO085 RVC 模式

### 2.1 硬件配置

**PS0 接 VCC**，上电即进入 RVC 模式，MSPM0 纯被动接收，无需初始化。

| 参数 | 值 |
|------|-----|
| 波特率 | 115200 |
| 帧长度 | 19字节 |
| 输出频率 | ~100Hz |
| 接线 | BNO085 TX → MSPM0 UART RX |
| 安装位置 | 底盘重心位置 |

### 2.2 RVC 帧格式

| 字节 | 内容 |
|------|------|
| [0-1] | 帧头：0xAA 0xAA |
| [2-3] | Yaw (int16, 小端, 0.01°) |
| [4-5] | Pitch (int16, 小端, 0.01°) |
| [6-7] | Roll (int16, 小端, 0.01°) |
| [8-13] | Accel X/Y/Z (int16, 小端, mg) |
| [14-17] | 保留 |
| [18] | 帧序号 |

### 2.3 横摆角速度计算

RVC 不输出原始角速度。对相邻帧 Yaw 角做微分 + 一阶低通滤波：

```c
float raw_gyro = (yaw_now - yaw_prev) / dt;           // dt ≈ 10ms (100Hz)
yaw_rate = alpha * raw_gyro + (1.0f - alpha) * yaw_rate;  // alpha = 0.25~0.35
```

> Yaw 角微分不存在万向节锁问题（仅微 Yaw，不涉及 Pitch/Roll）。8-12ms 延迟对前馈完全够用——车身转弯频率远低于 10Hz。

### 2.4 SHTP 备选方案

如果需要更低延迟或更高频率，可改用 SHTP 模式（PS0=GND），直接读取原生陀螺角速度。但代码复杂度增加，且需要双向通信。RVC 方案对底盘前馈已足够。

## 三、发送给云台的协议

### 3.1 帧格式

10 字节二进制帧，小端序，~100Hz：

| 偏移 | 大小 | 字段 | 说明 |
|------|------|------|------|
| 0 | 1 | 帧头1 | 0xAA |
| 1 | 1 | 帧头2 | 0x55 |
| 2 | 1 | cmd | 0x01 = 底盘状态 |
| 3-6 | 4 | yaw_rate | float32, 横摆角速度 (°/s) |
| 7-8 | 2 | seq | uint16, 包序号（溢出归零） |
| 9 | 1 | checksum | 字节[0..8] 的 XOR |

### 3.2 发送示例（伪代码）

```c
void chasis_fasong_yaw_rate(float yaw_rate, uint16_t seq) {
    uint8_t buf[10];
    buf[0] = 0xAA;
    buf[1] = 0x55;
    buf[2] = 0x01;
    memcpy(&buf[3], &yaw_rate, 4);  // float32, little-endian
    buf[7] = seq & 0xFF;
    buf[8] = (seq >> 8) & 0xFF;
    buf[9] = buf[0] ^ buf[1] ^ buf[2] ^ buf[3] ^ buf[4]
           ^ buf[5] ^ buf[6] ^ buf[7] ^ buf[8];
    uart_send(buf, 10);
}
```

### 3.3 物理连接

MSPM0 UART TX → 导电滑环 → STM32 UART4 RX (PA1)

## 四、差速转向平滑控制

### 4.1 目标

转弯时左右轮 PWM 不直接跳变，而是按设定的最大角加速度限制，生成平滑的横摆角速度上升曲线。

### 4.2 控制策略

```
目标角速度 → 角加速度限幅 → 梯形/S形曲线 → 左右轮差速PWM
                                      ↑
                              底盘IMU反馈实际角速度
                              超调/突变时抑制差速
```

1. 需要转弯时，系统按最大角加速度限制，生成平滑的角速度上升曲线
2. 左右轮 PWM 占空比梯形或 S 形渐进变化
3. 底盘 IMU 实时反馈实际角速度，检测超调或突变时立即抑制差速
4. 过弯姿态圆滑连贯，不给云台制造剧烈晃动

### 4.3 参数（由底盘同学调）

| 参数 | 说明 |
|------|------|
| 最大角加速度 | 限制转弯加速度，越大转弯越快但越剧烈 |
| 目标角速度 | 转弯时的目标偏航角速度 |
| PWM 差速上限 | 左右轮最大速度差 |
| 加速曲线类型 | 梯形 / S形 |

## 五、调试建议

### 5.1 单独调试底盘

断开云台功能，专注于差速转向平滑性：
1. 调整速度曲线参数
2. 用底盘 IMU 观察横摆角速度曲线
3. 确保连续平滑无折角

### 5.2 前馈数据验证

1. MSPM0 持续发送 yaw_rate 帧
2. 云台 STM32 串口1打印接收到的 yaw_rate 值
3. 手动转动底盘，确认 yaw_rate 符号和大小正确
4. 右转应为正值，左转应为负值（或反过来，取决于 IMU 安装方向）

### 5.3 故障排查

| 现象 | 可能原因 | 排查方法 |
|------|---------|---------|
| 云台收不到 yaw_rate | MSPM0 没发 / 滑环接触不良 | 示波器量 MSPM0 TX 引脚 |
| yaw_rate 方向反 | IMU 安装方向不对 | 调换 IMU 方向或软件取反 |
| yaw_rate 噪声大 | 微分滤波 alpha 太大 | 减小 alpha（更平滑） |
| 转弯时云台剧烈晃动 | 底盘角加速度太大 | 减小最大角加速度限制 |
