# 云台控制方案 — STM32F407VET6

## 一、方案总览

STM32F407VET6 作为云台主控，安装在云台平台上。控制两个 GM6020 无刷电机实现偏航(yaw)和俯仰(pitch)双轴瞄准，配合 maixcam2 视觉追踪和 BNO085 陀螺仪增稳。

### 核心控制律

```
电机输出 = 位置环(编码器) + 速度环(IMU角速度) + IMU阻尼 + 底盘前馈
```

- **位置环**：编码器角度做反馈（不漂移），视觉目标做输入
- **速度环**：IMU Gyro 原生角速度做反馈（响应快，无量化噪声）
- **IMU阻尼**：-Kd × gyro_rate，抵抗外部扰动（手推、颠簸、风阻）
- **底盘前馈**：-Kff × chassis_yaw_rate，车身转弯时同步补偿（后续接入）

### 硬件清单

| 部件 | 型号 | 接口 | 说明 |
|------|------|------|------|
| 主控 | STM32F407VET6 | — | 在云台平台上 |
| 偏航电机 | GM6020 | CAN, ID=1 | DIP 001, 反馈 0x205 |
| 俯仰电机 | GM6020 | CAN, ID=2 | DIP 010, 反馈 0x206 |
| 云台IMU | BNO085 | USART2, UART-SHTP 3Mbps | 云台板上直连，USART2 专用 |
| 视觉 | maixcam2 | USART3 | 云台板上直连 |
| 调试 | USB-TTL | USART1 | PA9/PA10 |

### 与底盘的接口

底盘 MSPM0G3507 通过滑环发来横摆角速度数据（10字节帧），接 STM32 UART4 RX。K_ff 初始为 0（关闭），云台部分独立调通后再接入。

| 5V | 相机 + BNO085 供电 |
| MSPM0 TX → UART4 RX | yaw_rate 数据 |

### 4.2 控制律

> **IMU 用在哪？** 云台IMU通过SHTP协议启用 Gyro Calibrated (0x01)，**只用角速度做速度环和阻尼**。位置环用编码器角度。
>
> **为什么不用IMU做位置环？** 不开磁力计时，BNO085的Yaw角靠陀螺积分，**会随时间漂移**（每分钟几度）。编码器是绝对式的，不漂移，多圈精确。所以：
> - **位置环 → 编码器角度**（不漂移，1kHz更新，GM6020直驱无间隙）
> - **速度环 → IMU Gyro**（原生角速度，无微分噪声，比编码器速度更平滑）
> - **阻尼 → IMU Gyro**（同上）

**偏航轴 (yaw) / 俯仰轴 (pitch) —— 结构相同：**
```
target_angle   = 视觉dx/dy累加更新                          // "靶心在哪"
encoder_angle  = motor_date[x].total_du                     // 编码器多圈角度(位置环反馈)
gyro_rate      = SHTP Gyro Calibrated 角速度                // IMU原生角速度(速度环+阻尼)

angle_err    = target_angle - encoder_angle                 // 位置环: 编码器做反馈
target_vel   = Kp_pos * angle_err
vel_err      = target_vel - gyro_rate                       // 速度环: IMU角速度做反馈
vel_pid_out  = Kp_vel*vel_err + Ki_vel*∫vel_err
damping      = -Kd_gyro * gyro_rate                         // IMU角速度做阻尼
ff           = -K_ff * chassis_yaw_rate                     // 底盘前馈(仅偏航,俯仰=0)
motor_volts  = vel_pid_out + damping + ff                   // 最终输出,限幅±25000
```

### 4.2.1 视觉+IMU+编码器 融合控制（第四阶段）

**核心架构：**
```
视觉 dx/dy（目标在哪里）
       ↓
换算成目标 yaw/roll 增量
       ↓
IMU姿态外环（云台现在朝哪里）
       ↓
目标速度
       ↓
GM6020编码器速度环（电机内层反馈）
       ↓
电机输出
```

**一句话：视觉定目标，IMU测姿态，编码器控电机**

**关键要点：**
- **视觉模块**：告诉云台目标在哪里（dx/dy）
- **BNO085**：告诉云台自己现在朝哪里、有没有被扰动
- **GM6020编码器**：继续做电机内层速度/角度反馈
- **不要完全替换位置环**：用混合模式，IU0/IU1开关可切换

**坐标系映射（实测）：**
- X轴偏航 → Yaw（开机Tare归零，不依赖地磁北）
- Y轴俯仰 → Roll（重力参考，比Yaw更可靠）

**控制律（融合模式）：**
```
视觉dx/dy → 更新 gimbal_x_target / gimbal_y_target
imu_yaw_relative = bno085_data.rotation.yaw - imu_yaw_offset
imu_roll_relative = bno085_data.rotation.roll - imu_roll_offset
gimbal_x_speed_target = PID_PositionLoop(&pid_x_pos, imu_yaw_relative, gimbal_x_target)
gimbal_y_speed_target = PID_PositionLoop(&pid_y_pos, imu_roll_relative, gimbal_y_target)
```

**关键坑：**
- **Yaw不要太相信地磁北**：金属结构干扰磁力计，用开机Tare归零
- **Roll=170.87要小心**：安装接近180°翻转，容易跨180°跳变
- **必须保留编码器速度环**：2ms内环不要换，IMU放外环
- **ShortestPath_Error**：角度环绕处理，避免±180°跳变
- **IU0/IU1开关**：可随时退回编码器位置环

**串口调参命令：**
```
IO0.0   - Yaw开机偏移（开机时记录）
IR0.0   - Roll开机偏移（开机时记录）
IU0     - 使用编码器位置环（默认）
IU1     - 使用IMU姿态外环
```

### 4.3 时序调度

```
TIM2 ISR (每1ms触发):

time_ms++

每2ms (500Hz 速度环):
├── 读取云台IMU角速度 gyro_rate_yaw / gyro_rate_pitch (USART2 3Mbps SHTP Gyro Calibrated)
├── 读取底盘yaw_rate chassis_yaw_rate (UART4, 初始=0)
├── 偏航: 速度PI + IMU阻尼 + 底盘前馈 → yaw_voltage
├── 俯仰: 速度PI + IMU阻尼            → pitch_voltage
├── 限幅 ±25000
└── CAN_Transmit(yaw_voltage, pitch_voltage, 0, 0, Voltage)

每10ms (100Hz 位置环):
├── 读取编码器角度 motor_date[0/1].total_du
├── 偏航: 位置P → target_vel_yaw
├── 俯仰: 位置P → target_vel_pitch
└── time_ms = 0

主循环 (最低优先级):
├── 检查 bn085_ready → BNO085_Process → 更新IMU角速度
├── 检查 chassis_frame_ready → 解析底盘yaw_rate (K_ff=0时可跳过)
├── 检查 cam_frame_ready → 解析视觉帧 → 更新目标角度
└── 检查 dbg_cmd_ready → 调试指令
```

### 4.4 数据速率

| 数据源 | 速率 | 用途 | 来源 |
|--------|------|------|------|
| BNO085 Gyro Calibrated | ~100Hz | 速度环反馈+阻尼(2ms) | USART2 UART-SHTP 3Mbps |
| GM6020 编码器角度 | ~1kHz | 位置环反馈(10ms) | CAN反馈 |
| 底盘yaw_rate | ~100Hz | 前馈(2ms) | MSPM0→UART4 |
| maixcam2 dx/dy | ~20-50Hz | 目标角度更新 | USART3 |
| CAN电压输出 | 500Hz | 电机执行 | CAN1 |

## 五、BNO085 云台IMU — SHTP 协议

### 5.1 硬件配置

BNO085 使用 **UART-SHTP 模式**，USART2 双向通信（PA2=TX, PA3=RX），串口速率固定为 **3Mbps**。

> USART2 专门给 BNO085 使用，原来接在 USART2 的 HA8-U35-M 舵机不再使用。后续代码迁移时需要删除 `fashion_star_uart_servo` 初始化、收发中断和 Y 轴舵机控制逻辑。

硬件协议选择脚按实际 BNO085 模块要求配置为 UART-SHTP；上电后先用 Product ID Request 验证通信。如果无法收到回包，优先检查协议选择脚、复位时序、3Mbps 波特率误差和 TX/RX 交叉连接。

### 5.2 坐标系映射（实测）

> ⚠️ 以下映射关系基于本项目云台的BNO085安装方式，不同安装方式需要重新测量。

**测试数据格式（串口1输出）：**
```
Yaw: 173.35 Pitch:- 21.30 Roll: 170.87 Gx:- 11.1 Gy:-  3.2 Gz:   6.9 acc:0
```

**云台轴 ↔ BNO085 轴对应关系：**

| 云台轴 | 运动方式 | BNO085姿态 | BNO085角速度 | 说明 |
|--------|----------|------------|--------------|------|
| **X轴（偏航）** | 水平转动 | **Yaw** | **Gz** | 零度=地磁北，西正东负 |
| **Y轴（俯仰）** | 上下摆动 | **Roll** | **Gx** | 俯角(低头)=正，仰角(抬头)=负 |

**关键发现：**
- 云台俯仰对应BNO085的**Roll轴**，不是Pitch轴
- 这是由BNO085在云台上的实际安装方向决定的

**阻尼项应用：**
```c
// X轴偏航阻尼（使用Gz）
float x_damping = Kd_gyro * bno085_data.gyro.z;
yaw_speed_target = yaw_speed_target - x_damping;

// Y轴俯仰阻尼（使用Gx）
float y_damping = Kd_gyro * bno085_data.gyro.x;
pitch_speed_target = pitch_speed_target - y_damping;
```

### 5.3 启用的报告

只需 **Gyro Calibrated (Report ID 0x01)**，提供原生角速度。不需要 Rotation Vector（位置环用编码器）。

### 5.4 SHTP 帧格式

**帧头（4字节）：**

| 字节 | 内容 |
|------|------|
| [0-1] | Payload Length (uint16, 小端) |
| [2] | Channel |
| [3] | Sequence Number |

**Gyro Calibrated 报告（Report ID 0x01）：**

| 字节 | 内容 |
|------|------|
| [0] | Report ID (0x01) |
| [1-2] | Gyro X (int16, 小端, 0.01 rad/s ≈ 0.573°/s) |
| [3-4] | Gyro Y (int16, 小端) |
| [5-6] | Gyro Z (int16, 小端) — 偏航角速度 |
| [7-12] | Accel X/Y/Z (int16, 小端, mg) |
| [13] | Calibration Accuracy (0-3) |

### 5.5 初始化步骤

1. 发送 Product ID Request (0xF9) → 确认设备在线
2. 发送 Set Feature Report → 启用 Gyro Calibrated (sensor ID 0x01)，设定报告间隔（如 10ms = 100Hz）
3. 收到确认后，BNO085 开始推送角速度数据

### 5.6 RVC 备选方案

当前正式方案不采用 RVC。RVC 只作为临时排障手段，因为 RVC 是 115200 被动输出姿态角，不提供本方案需要的 Gyro Calibrated 原生角速度；如果退回 RVC，控制律需要重新评估，不能直接套用本方案的速度环。

## 六、视觉协议 (maixcam2, USART3) — 待视觉组最终确认

视觉相关代码和通信字段由视觉组继续更新，当前云台侧先只锁定接口原则：

- 物理接口：USART3 RX，默认 115200，DMA NORMAL + IDLE。
- 云台侧输入需求：至少需要 `cmd`、`dx`、`dy` 三个字段。
- 如果视觉组提供目标框宽度、距离、置信度等字段，云台侧再接入动态 `kv`、目标丢失保护和调试打印。
- 在视觉协议最终版确认前，`YunTai_ProcessCam()` 只预留解析入口，不把字段格式写死进控制模块。

下面是当前建议版协议，后续以视觉组最终代码为准：

| 字节 | 内容 |
|------|------|
| [0] | 帧头1：0xAA |
| [1] | 帧头2：0xBB |
| [2] | cmd：0=扫描，1=追踪 |
| [3-4] | dx（int16，小端）：目标偏离图像中心X像素（已滤波+逆透视校正） |
| [5-6] | dy（int16，小端）：目标偏离图像中心Y像素 |
| [7-8] | w（uint16，小端）：框的像素宽度（用于距离估算） |
| [9] | 校验和：(cmd + dx低 + dx高 + dy低 + dy高 + w低 + w高) & 0xFF |

> maixcam2 端负责卡尔曼滤波和逆透视变换，STM32 收到的 dx/dy 已经是干净数据。
> w 字段用于估算目标距离，进而动态调整放大系数。详见 [视觉方案.md](视觉方案.md)。

## 七、底盘通信协议 (UART4)

底盘 MSPM0G3507 发送的 10 字节二进制帧，小端序：

| 偏移 | 大小 | 字段 | 说明 |
|------|------|------|------|
| 0 | 1 | 帧头1 | 0xAA |
| 1 | 1 | 帧头2 | 0x55 |
| 2 | 1 | cmd | 0x01 = 底盘状态 |
| 3-6 | 4 | yaw_rate | float32, 横摆角速度 (°/s) |
| 7-8 | 2 | seq | uint16, 包序号 |
| 9 | 1 | checksum | 字节[0..8] 的 XOR |

**STM32侧：** DMA CIRCULAR接收 + IDLE中断，搜帧头 → XOR校验 → 提取yaw_rate。超时200ms无有效帧 → 衰减yaw_rate到零。

> K_ff 初始为 0，此协议在云台部分独立调试时可暂不启用。

## 八、GM6020 CAN 协议

| 项目 | 标识符 |
|------|--------|
| 控制报文（电压模式） | 0x1FF |
| 反馈报文（电机1=偏航, ID=1） | 0x205 |
| 反馈报文（电机2=俯仰, ID=2） | 0x206 |

**控制帧 (0x1FF, 8字节):**

| 字节 | 内容 |
|------|------|
| DATA[0-1] | 电机1(偏航) 电压 (int16, 大端, ±25000) |
| DATA[2-3] | 电机2(俯仰) 电压 (int16, 大端, ±25000) |
| DATA[4-5] | 电机3 (未使用, 发0) |
| DATA[6-7] | 电机4 (未使用, 发0) |

**反馈帧 (0x205/0x206, 8字节):**

| 字节 | 内容 |
|------|------|
| DATA[0-1] | 机械角度 (0~8191, 大端) |
| DATA[2-3] | 转速 (rpm, 大端有符号) |
| DATA[4-5] | 转矩电流 (大端有符号) |
| DATA[6] | 温度 (℃) |
| DATA[7] | 保留 |

**DIP开关设置：**
- 偏航电机：ID=1 (DIP 1-3: 001) → CAN反馈ID=0x205
- 俯仰电机：ID=2 (DIP 1-3: 010) → CAN反馈ID=0x206
- **只拨一个电机的DIP第4位**（ON=120Ω），两个都拨=60Ω并联偏大

## 九、代码架构

### 9.0 当前代码迁移目标

当前工程仍是早期版本：

- X 轴：GM6020，通过 CAN 控制。
- Y 轴：HA8-U35-M 舵机，通过 USART2 控制。
- USART2：被舵机驱动占用。
- 视觉：USART3 DMA 接收，协议仍可能继续变化。
- BNO085、UART4 底盘前馈、双 GM6020 云台控制模块尚未接入。

目标工程改为：

- 偏航轴：GM6020，`motor_date[0]`，CAN 反馈 ID `0x205`。
- 俯仰轴：GM6020，`motor_date[1]`，CAN 反馈 ID `0x206`。
- USART2：只连接 BNO085，UART-SHTP 3Mbps。
- USART3：只接 maixcam2，等视觉组最终协议后再固定解析。
- USART1：调试和参数命令。
- UART4：底盘 yaw_rate 前馈，后续接入，`K_ff` 初始为 0。

迁移原则：

1. 不再使用 `fashion_star_uart_servo/`，也不再调用 `FSUS_INIT()`、`FSUS_SetServoAngle()`、`User_Uart_Init(&huart2)`。
2. 先让两个 GM6020 都能独立收反馈、发电压、做速度测试，再接入完整云台控制律。
3. 视觉协议未最终确认前，只保留视觉处理接口，不把动态距离、`w` 字段和 `kv` 计算强行写死。
4. BNO085 先完成 3Mbps SHTP 通信和角速度打印，再把 gyro 接入速度环和阻尼。
5. 底盘 UART4 前馈最后接入，避免影响云台本体调试。

### 9.0.1 推荐实施顺序

**阶段1：双 GM6020 基础控制**

- 修改 CubeMX/CMake/Keil 工程，移除舵机相关编译项。
- `main.c` 删除舵机初始化和 Y 轴舵机定时发送。
- `gm6020_can.c` 确认 0x205/0x206 都能解析，给 yaw/pitch 分别设置零点。
- 调试命令增加单轴测试：`YT 100` 测 yaw 速度，`PT 100` 测 pitch 速度。
- CAN 输出改为 `CAN_Transmit(yaw_voltage, pitch_voltage, 0, 0, Voltage)`。

**阶段2：云台控制模块化**

- 新建 `User/yuntai_kongzhi/`。
- 把 `target_angle`、`target_velocity`、PID 参数、限幅、死区、调试命令从 `main.c` 移到模块内。
- TIM2 中断只调用 `YunTai_ControlLoop()`，主循环只做 `YunTai_ProcessCam()` 和 `YunTai_ProcessDebug()`。

**阶段3：BNO085 接入**

- USART2 改为 3Mbps TX+RX，RX DMA Circular + IDLE。
- 新建 `User/bno085/`，先完成 Product ID Request 和 Gyro Calibrated 报告解析。
- 串口1打印 `gyro_rate_yaw`、`gyro_rate_pitch`，确认符号和安装方向。
- 确认稳定后，将速度环反馈从 GM6020 编码器速度切换到 BNO085 gyro。

**阶段4：视觉协议落地**

- 等视觉组提供最终帧格式后，更新 `YunTai_ProcessCam()`。
- 如果有 `w` 或距离字段，再加入动态 `kv` 和目标丢失保护。
- 没有距离字段时，先使用固定 `kv_base` 和单次增量限幅。

**阶段5：底盘前馈**

- 新增 UART4 + DMA Circular + IDLE。
- 接入 `chassis_comm` 解析 yaw_rate。
- `K_ff` 默认 0，先只打印，不参与控制；确认方向后再逐步加前馈。

## 十二、CubeMX 配置要点

1. **USART2** (云台IMU BNO085 SHTP)
   - Mode: Asynchronous → **TX+RX**
   - Pin: PA2=TX, PA3=RX
   - Baud: **3000000**, 8N1
   - DMA: USART2_RX → **DMA1 Stream5, Channel 4, Mode: Circular**
   - NVIC: USART2 global interrupt ✅ + DMA1 stream5 global interrupt ✅
   - USART2 不再连接舵机，不再启用 `fashion_star_uart_servo` 的中断接收

2. **UART4** (底盘通信)
   - 新增外设，PA1=UART4_RX, PA0=UART4_TX
   - Mode: Asynchronous → RX Only
   - Baud: 115200, 8N1
   - DMA: UART4_RX → **DMA1 Stream2, Channel 4, Mode: Circular**
   - NVIC: UART4 global interrupt ✅ + DMA1 stream2 global interrupt ✅

3. **USART3** (视觉)
   - DMA: DMA1 Stream1, Channel 4, Mode: Normal
   - NVIC: USART3 global interrupt ✅
   - 协议字段等待视觉组最终版本确认

4. **生成代码后**，在 stm32f4xx_it.c 接入：
   - USART2_IRQHandler → `BNO085_IRQHandler()`
   - UART4_IRQHandler → `Chassis_IRQHandler()`

## 十四、注意事项

### 硬件

1. GM6020 需要 24V 供电，两个电机共用（经滑环）
2. CAN 终端电阻：**只拨一个电机的DIP第4位**（两个都拨=60Ω并联偏大）
3. BNO085 使用 **UART-SHTP 3Mbps**，协议选择脚按模块手册配置，上电后用 Product ID Request 验证
4. STM32在云台上，CAN/USART2/USART3全部板上直连，不过滑环
5. 滑环仅走4根线：24V、5V、MSPM0 TX→UART4 RX、GND
6. GND 必须作为独立信号地走滑环，不依赖结构件接地
7. 滑环云台侧入口并联大容量电容（≥470μF），吸收接触波动

### 编译

- **Keil MDK**: 打开 `MDK-ARM/hc.uvprojx`，Rebuild (F7)
- **CMake + Ninja**: `cd build/Debug && cmake --build .`
- **clangd报错**: 删除 `build/Debug/.cache` 后重启IDE

### 调参顺序

1. **确认数据通**：串口1打印 `?`，确认 IMU 角速度正常、编码器角度正常
2. **P-only位置环**（Ki_vel=0, Kd_gyro=0, K_ff=0），调 Kp_pos 让云台跟上目标
3. **加P-only速度环**，调 Kp_vel 让跟踪平滑不振荡
4. **加 Kd_gyro 阻尼**（从0.5起），手动晃动云台验证增稳效果
5. **加 K_ff 前馈**（从0.3起），转动底盘验证补偿（需要底盘MSPM0配合）
6. **最后加 Ki_vel**（从0.2起）消除稳态误差
7. 调参前确认：IMU角速度符号与电机转向一致

### 故障排查

| 现象 | 可能原因 | 排查方法 |
|------|---------|---------|
| 某个电机不动 | CAN不通 / ID设错 / DIP开关 | 串口打印motor_date温度，非0说明CAN通 |
| 两个电机都不动 | CAN终端电阻配置错 | 确认只拨了一个电机DIP第4位 |
| IMU数据全零 | USART2不是3Mbps / DMA没配CIRCULAR / SHTP模式脚错误 / TX RX接反 | 发0xF9看有无回包 |
| 底盘前馈无效 | MSPM0没发数据 / UART4配置错 / K_ff=0 | 串口打印chassis_yaw_rate |
| 阻尼方向反 | Kd_gyro极性不对 | 顺时针晃云台→gyro_rate正→电机应逆时针补偿 |
| 前馈方向反 | K_ff极性不对 | 右转底盘→yaw_rate正→偏航电机应左转 |
| 光斑抖动 | Kp_vel太大 / Kd_gyro太小 | 减Kp_vel或增Kd_gyro |
| 光斑稳态偏移 | Ki_vel太小 | 增大Ki_vel |
| 过弯漂移 | K_ff太小 | 增大K_ff |
